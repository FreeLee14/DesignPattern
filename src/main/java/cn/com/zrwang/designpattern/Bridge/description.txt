Bridge（桥接模式）：当前若存在复合功能而需要多继承时，也可以考虑桥接模式，其一般应用在有两个非常强的变化维度时，有时存在多于2个维度的变化方向，
                  也可以考虑桥接模式，各个维度都需要抽象出一个基类，同时选择一个维度作为主维度，这个主维度基类包含依赖其他各个维度的基类
                  的指针（在Java中可以称为引用）
                  例如：一个平台，存在着平台方向功能和业务方向功能，平台方向功能有：登陆、连接、传送信息，业务方面的功能是：发出声音、绘画、书写文本
                  平台方向的功能是需要和业务方面的功能进行搭配组合形成不同版本级别的功能。其中平台方向功能需要依赖业务方面的不同功能的组合
                  以登陆功能为例，一个登陆过程当中，在一个简易版本系统的登陆：会包含声音，绘画。而一个专业版本系统的登陆：会包含声音、绘画、书写文本。

                  桥接模式与装饰器模式的区别在于，桥接模式是通过抽象类依赖，完成静态编译阶段的功能组合，而装饰器模式是完全的动态功能组合。
