AbstractFactory (抽象工厂模式)：抽象工厂模式用于生产一系列"相互依赖的对象"，普通的工厂模式是一个工厂生产一种对象，而抽象工厂模式是一个工厂
                              生产属于某个系列的多种对象。其实我们可以把普通工厂模式看作是抽象工厂模式的子类。
                              例如我们要封装数据库连接的工具包，连接并执行sql语句的过程，需要创建连接对象，会话对象，以及操作对象，而数据库
                              的种类又分为 MySql、SqlServer、Oracle，此时我们需要使用抽象工厂方法进行创建
